# 교회 SEO/GEO 진단 스킬

## 개요

교회 및 기독교 기관 웹사이트를 SEO, AEO, GEO, 네이버 검색 최적화 관점에서 종합 진단하고, 실행 가능한 개선안을 제시한다.

## 입력 방식

두 가지 입력을 지원한다. 둘 다 제공되면 더 정밀한 진단이 가능하다.

1. **URL 입력** — 교회 웹사이트 주소를 입력받아 실제 페이지를 분석
2. **키워드 입력** — 교회가 노출되길 원하는 검색 키워드 (예: "강남 교회", "주일예배 시간")

## 진단 절차

### 1단계: 정보 수집

**URL이 제공된 경우:**
- `web_fetch`로 메인 페이지 HTML을 가져온다
- 메타태그(title, description, og:tags), 구조화 데이터(JSON-LD), 모바일 뷰포트, canonical 태그를 확인한다
- 주요 하위 페이지(소개, 예배안내, 오시는길, 교회소식)도 가능하면 확인한다

**키워드가 제공된 경우:**
- `web_search`로 해당 키워드의 구글 검색 결과를 확인한다
- 교회가 검색 결과에 노출되는 위치와 형태를 파악한다
- 경쟁 교회/기관의 노출 상태를 비교한다

**ListeningMind MCP가 연결된 경우:**
- `keyword_info`(data_type=all)로 해당 키워드의 검색량, 인구통계를 확인한다
- `path_finder`로 검색자의 탐색 경로를 추적한다
- 단, `f_ai_overview` 플래그는 실제 구글 결과와 다를 수 있으므로 참고용으로만 사용한다

### 2단계: 영역별 진단

각 영역의 상세 체크리스트는 references/ 폴더에 있다. 진단 시 해당 파일을 읽고 항목별로 점검한다.

| 영역 | 참조 파일 | 핵심 점검 포인트 |
|------|-----------|-----------------|
| 기본 SEO | `references/seo-checklist.md` | 메타태그, 구조화 데이터, 사이트맵, 모바일, 속도, 접근성 |
| AEO/GEO | `references/geo-aeo-checklist.md` | FAQ 구조, 질문형 콘텐츠, 인용 가능성, E-E-A-T 신호 |
| 네이버 | `references/naver-checklist.md` | 네이버 웹마스터, 스마트플레이스, 블로그 연동, 검색어 노출 |

### 3단계: 보고서 생성

진단 결과를 종합해 보고서를 생성한다.

**보고서 구조:**

```
# [교회명] SEO/GEO 진단 보고서

## 1. 진단 개요
- 진단 대상, 진단 일시, 입력 키워드

## 2. 종합 점수
- 영역별 점수 (기본 SEO / AEO / GEO / 네이버) — 각 100점 만점
- 종합 등급 (A/B/C/D/F)

## 3. 영역별 진단 결과

### 3.1 기본 SEO
- 현황 요약
- 발견된 문제 (심각도: 높음/중간/낮음)
- 개선 권고사항

### 3.2 AEO (AI 답변 최적화)
- 현황 요약
- 발견된 문제
- 개선 권고사항

### 3.3 GEO (생성형 검색 최적화)
- 현황 요약
- 발견된 문제
- 개선 권고사항

### 3.4 네이버 검색 최적화
- 현황 요약
- 발견된 문제
- 개선 권고사항

## 4. 우선순위 액션 플랜
- 즉시 실행 (1주 이내)
- 단기 개선 (1개월 이내)
- 중장기 과제 (3개월 이내)

## 5. 참고사항
- 진단의 한계, 추가 확인이 필요한 항목
```

## 출력 형식

사용자의 요청에 따라 유연하게 대응한다.

- **기본**: 대화 내 텍스트로 보고서 제공
- **"파일로 줘", "문서로 만들어 줘"**: HTML 또는 Word(.docx) 파일로 생성
- **"간단히 봐 줘", "요약만"**: 종합 점수와 상위 3개 개선사항만 제공

## 점수 산정 기준

각 영역 100점 만점. 체크리스트 항목별 가중치를 적용한다.

- **높음 가중치 항목** 미충족: -15점
- **중간 가중치 항목** 미충족: -8점
- **낮음 가중치 항목** 미충족: -3점

종합 등급:
- A (90-100): 검색 최적화 우수
- B (75-89): 양호, 부분 개선 필요
- C (60-74): 보통, 체계적 개선 필요
- D (40-59): 미흡, 상당한 개선 필요
- F (0-39): 검색 최적화 미비

## 톤과 스타일

- 보고서는 비전문가(목사, 교회 행정 담당자)가 읽는다는 전제로 작성한다
- 기술 용어에는 짧은 설명을 괄호로 병기한다 (예: "구조화 데이터(검색엔진이 읽는 기계용 정보)")
- 문제 지적보다 개선 방향에 무게를 둔다
- 한국어 하우스 스타일을 적용한다: 합니다체, 부사 절제, 짧은 문장, 선언적 시작 금지

## 주의사항

- web_fetch로 가져온 HTML은 렌더링 후 결과가 아닌 원본 소스이므로, JavaScript로 동적 생성되는 콘텐츠는 확인이 제한된다
- 네이버 검색 결과는 web_search로 직접 확인할 수 없으므로, 네이버 영역은 구조적 권고(스마트플레이스 등록 여부, 블로그 운영 상태 등) 중심으로 진단한다
- 페이지 로딩 속도는 직접 측정할 수 없으므로, HTML 크기와 외부 리소스 수로 간접 추정한다
- 검증할 수 없는 수치나 주장은 포함하지 않는다
